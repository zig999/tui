# Template: delivery-gate (embedded in tc-XX-delivery.md)

> Layer: semi-permanent — embedded inside `$SESSION_DIR/delivery/$ORCH_TASK_ID-delivery.md`
> Generated by: u-fe-developer | u-be-developer
> Consumed by: u-fe-qa | u-be-qa (pre-test-gate gate)

The delivery-gate is a YAML block placed **at the very top of `tc-XX-delivery.md`**, before the Markdown body. The QA Agent reads this block first. If `qa_ready: false`, QA stops immediately and returns a blocked report — it does not run the test suite.

**Rules:**
- `tests.all_pass` reflects the Developer's last local run — QA will re-run and verify
- `acceptance_criteria.uncovered` lists the exact criterion text of any unimplemented AC
- `spec_divergences.items` are required when `spec_divergences.count > 0`
- `qa_ready: false` is only valid when `status: implemented_with_caveats` AND there is a structural reason QA cannot proceed (e.g., test command broken, environment missing)
- Do NOT omit this block — QA will treat a missing gate as `qa_ready: false`

---

```yaml
# delivery-gate
task_id: TC-XX
layer: semi-permanent
delivered_by: u-fe-developer | u-be-developer
timestamp: <YYYY-MM-DDTHH:MM:SSZ>

status: implemented | implemented_with_caveats

tests:
  command: <exact test command — e.g. npx vitest run>
  last_local_run: passed | failed
  total: <int>
  passed: <int>
  failed: <int>                          # 0 when last_local_run: passed

acceptance_criteria:
  total: <int>
  covered: <int>
  uncovered:                             # empty list when all covered
    - <exact criterion text>

spec_divergences:
  count: <int>
  items:                                 # empty list when count 0
    - id: DIV-001
      description: <single objective sentence>
      type: necessary | accidental       # necessary = spec gap; accidental = Developer error

tech_debt:
  count: <int>

# observability: (required when CLAUDE.md declares observability_required: true)
# observability:
#   structured_logging: true | false
#   trace_id_propagated: true | false
#   health_endpoint_present: true | false
#   metrics_exposed: true | false

# dependency_audit: (required when CLAUDE.md declares dependency_audit: true)
# dependency_audit:
#   command: <exact audit command — e.g. npm audit --json | jq '.metadata.vulnerabilities'>
#   vulnerabilities_critical: <int>
#   vulnerabilities_high: <int>
#   vulnerabilities_medium: <int>

# nfr_results: (required when task_contract has non_functional_requirements)
# nfr_results:
#   - type: latency_p99_ms
#     threshold: 200
#     measured: <int>
#     passed: true | false
#     command_run: <exact command executed>

qa_ready: true | false
qa_notes: <optional — what QA must pay special attention to>
```

## Usage in tc-XX-delivery.md

The delivery file must start with the gate block in a YAML code fence, followed by the Markdown body:

~~~markdown
```yaml
# delivery-gate
task_id: TC-23
layer: semi-permanent
delivered_by: u-fe-developer
timestamp: 2026-04-08T17:00:00Z

status: implemented

tests:
  command: npx vitest run
  last_local_run: passed
  total: 12
  passed: 12
  failed: 0

acceptance_criteria:
  total: 3
  covered: 3
  uncovered: []

spec_divergences:
  count: 0
  items: []

tech_debt:
  count: 1

qa_ready: true
qa_notes: Pay attention to the optimistic update in useUserProfile — rollback on 409 is tested but complex.
```

# Delivery: TC-23 — Edit user profile

**Date:** 2026-04-08
**Layer:** semi-permanent
**Status:** Implemented
...
~~~

## QA behavior on gate read

| Gate condition | QA action |
|---|---|
| `qa_ready: false` | Return blocked-report immediately — do not run tests |
| `tests.last_local_run: failed` | Flag to Orchestrator — Developer must fix before QA |
| `acceptance_criteria.uncovered` non-empty | Log each uncovered AC as Quality BUG (High) before running tests |
| `spec_divergences.count > 0` | Read items, classify as necessary or accidental per Definition of Done rules |
| Gate block missing | Treat as `qa_ready: false` — request Developer to add the gate |
