## 1. Purpose and Responsibilities

Renders a styled native `<input>` primitive: a single-line box-bordered text field consuming semantic tokens (`border-border`, `bg-background`, `text-foreground`, `placeholder:text-muted-foreground`), with reactive styling for the `disabled` and `aria-invalid` native attributes.

**Out of scope for this component:** label rendering, helper/error text, validation logic, and error-message display â€” these compose from separate primitives around this one (see MIGRATION note in source: the previous implementation's `$` prompt-prefix and its internal loading/error/success glyph state machine were dropped in favor of the standard `aria-invalid` contract). No file-input-specific reset styling is included (no file input variant in this kit).

## 4. Component States

> No internal state (no `useState`/`useReducer`). The component does not set `disabled` or `aria-invalid` itself â€” both are native attributes the consumer passes through `...props`; the rows below document the CSS reaction the component's stylesheet applies when the consumer sets them.

| State | Trigger | Visual change | Interactivity |
|-------|---------|----------------|----------------|
| `idle` | initial render, no `disabled`/`aria-invalid` set | base classes only (`border-border`, no override) | full |
| `invalid` | consumer sets `aria-invalid="true"` (or `true` boolean) | `aria-invalid:border-destructive` â€” border switches to the destructive token | full (field remains interactive) |
| `disabled` | consumer sets `disabled` | `disabled:cursor-not-allowed disabled:opacity-50` | none |

## 8. BDD Scenarios

### Default render

```
Given the component receives a placeholder and no other props
When it mounts
Then it renders a native <input> with base token classes and no error/disabled styling
```

### Invalid state

```
Given aria-invalid is set to true
When it mounts
Then the input's border switches to the destructive token (aria-invalid:border-destructive) while remaining focusable and editable
```

### Keyboard navigation

```
Given the input is focused
When the user types characters
Then the value updates, onChange fires for each keystroke, and the focus-visible ring (focus-visible:ring-ring) remains visible while focused
```
