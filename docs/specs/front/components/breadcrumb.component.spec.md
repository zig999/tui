## 1. Purpose and Responsibilities

Renders a navigational trail: a `<nav aria-label="breadcrumb">` wrapping an `<ol>` list built from the `items` array. For each item, the component computes whether it is the last entry (`index === items.length - 1`) and whether it carries an `href`, then renders one of three forms — current-page span, linked item (via the internal `Link` component), or plain text span — followed by an `aria-hidden` `/` separator between items (never after the last one).

**Out of scope for this component:** truncation/collapsing of long trails (no "..." condensation for many items), routing/navigation logic (delegated entirely to the internal `Link` component for `href` items), and stable per-item identity — list items are keyed by array `index`, not by `label`/`href`, so reordering the `items` array is not guaranteed to preserve per-item React state.

---

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Showing the hierarchical location of the current page/view as a trail of ancestor links plus the current page | There is only a single navigational link with no hierarchy to show → use `Link` instead |

## 3. Props Contract

```ts
// BreadcrumbItem is declared locally in breadcrumb.types.ts (not imported from a shared types module)
type BreadcrumbItem = string | { label: string; href?: string };
```

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `items` | `BreadcrumbItem[]` (`string \| { label: string; href?: string }`) | yes | — | Ordered trail of items. A plain `string` item renders as label-only text (never a link, even if not the last item). An object item renders as a link when it has `href` and is not the last item. |
| `className` | `string` | no | `undefined` | Merged via `cn()` onto the `<nav>` root element. |

Note: `BreadcrumbProps` is a plain interface (does not extend `ComponentProps<"nav">`) — the component destructures only `{ items, className }` and does not spread any native attribute passthrough onto the `<nav>`.

## 4. Component States

> No internal `useState`/`useReducer`. The rows below are state-like branches the component computes per list item from `index` and item shape (position/`href` presence) on every render — not a single component-wide state.

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `idle` (trail render) | initial render | renders `<nav aria-label="breadcrumb"><ol class="flex flex-wrap items-center text-xs">` with one `<li>` per item and a `/` separator between items | full (per sub-item, see below) |
| `current-item` | `index === items.length - 1` (last item, regardless of `href`) | renders `<span aria-current="page" className="text-primary">{label}</span>` — never a link, even if the last item is an object with `href` | none — not focusable |
| `linked-item` | not last, item is an object with a truthy `href` | renders the internal `Link` component: `<Link href={href} className="text-xs text-foreground">{label}</Link>` | full — focusable/clickable anchor |
| `text-item` | not last, and (item is a `string` OR item is an object with no `href`) | renders `<span className="text-muted-foreground">{label}</span>` | none — not focusable |

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Pass the last item as the current page (with or without `href` — it always renders as non-link text) | Expect the last item to render as a link even if it has an `href` — the component forces it to `<span aria-current="page">` |
| Use plain strings for items that should never be links | Rely on array index as a stable per-item key across reorders — items are keyed by `index` |

## 8. BDD Scenarios

### Default render

```
Given items = [{ label: "Home", href: "#" }, { label: "Projects", href: "#" }, "Current page"]
When the component mounts
Then it renders two Link items ("Home", "Projects") each followed by an aria-hidden "/" separator,
and a final <span aria-current="page"> "Current page" with no trailing separator
```

### No-links state (all text items)

```
Given items = ["Home", "Projects", "Current page"] (all plain strings)
When the component mounts
Then "Home" and "Projects" render as plain <span className="text-muted-foreground"> (no anchor),
and "Current page" renders as <span aria-current="page"> — no item in the trail is clickable
```

### Keyboard navigation

```
Given items include at least one non-last object item with href (a linked-item)
When the user presses Tab
Then focus moves to the underlying <a> rendered by the internal Link component for that item,
while current-item and text-item spans are skipped (not focusable, no tabIndex)
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | The `<nav>` has a fixed `aria-label="breadcrumb"`; the current page is marked via `aria-current="page"` on its `<span>` |
| Keyboard | Only `linked-item` entries are focusable/activatable (native `<a>` via the internal `Link`, Enter to activate); `current-item` and `text-item` spans are not focusable |
| Focus management | No custom focus trapping or redirection — focus order follows DOM order through the `<ol>` |
| ARIA states | `aria-current="page"` on the last item's `<span>`; `aria-hidden="true"` on every `/` separator span so it is not announced by assistive tech |
