## 1. Purpose and Responsibilities

Compound component wrapping `@radix-ui/react-dialog` (`^1.1.19`). Renders a modal dialog: a full-screen scrim overlay plus a centered panel that traps focus and blocks interaction with the rest of the page until dismissed.

Exported parts:
- `Dialog` — re-exported directly as `DialogPrimitive.Root` (no local wrapper). Owns the open/closed state.
- `DialogTrigger` — re-exported directly as `DialogPrimitive.Trigger` (no local wrapper).
- `DialogPortal` — re-exported directly as `DialogPrimitive.Portal` (no local wrapper).
- `DialogClose` — re-exported directly as `DialogPrimitive.Close` (no local wrapper). Standalone close trigger for use anywhere inside the dialog tree (e.g. a footer "Cancel" button).
- `DialogOverlay` — local wrapper around `DialogPrimitive.Overlay`; applies `fixed inset-0 z-50 bg-black/60`.
- `DialogContent` — local wrapper around `DialogPrimitive.Content`. This is the composition root: it renders `DialogPortal` containing `DialogOverlay` (no props passed) followed by `DialogPrimitive.Content` (fixed, centered, `max-w-md`, bordered `surface` panel), followed by `children`, followed by a **hardcoded** `DialogPrimitive.Close` icon button (`X` from `lucide-react`, `size={16}`, top-right absolute position, with an `sr-only` "Fechar" label).
- `DialogHeader` — local, unstyled `<div>` layout wrapper (`mb-4 flex items-center gap-3`). No Radix primitive underneath.
- `DialogFooter` — local, unstyled `<div>` layout wrapper (`mt-6 flex justify-end gap-2`). No Radix primitive underneath.
- `DialogTitle` — local wrapper around `DialogPrimitive.Title` (renders `<h2>`); applies `font-semibold tracking-wider text-accent uppercase`.
- `DialogDescription` — local wrapper around `DialogPrimitive.Description` (renders `<p>`); applies `text-sm text-muted-foreground`.

`DialogHeader`/`DialogFooter`/`DialogTitle`/`DialogDescription` are meant to be composed as `children` of `DialogContent`.

**Out of scope for this component:**
- Rendering a custom overlay — `DialogContent` always mounts its own `DialogOverlay` with no props forwarded to it; the exported `DialogOverlay`'s `className`/passthrough props are unreachable through the normal `DialogContent` composition path (only reachable by hand-building a `DialogContent`-equivalent).
- Configuring or hiding the built-in close (X) button — it is always rendered by `DialogContent`, with a fixed `className` and fixed "Fechar" label; no prop disables or restyles it independently.
- Entry/exit animation — no transition/animation classes are present in this wrapper. Radix sets `data-state="open" | "closed"` on `Content`/`Overlay`/`Trigger` that a consumer could target with custom CSS, but no such CSS exists here.

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Confirming a destructive or high-consequence action, or presenting a focused task/short form that must block interaction with the rest of the page until dismissed | Showing a transient, non-blocking status message → use `sonner`'s `<Toaster />` instead |
| | Showing a short contextual hint tied to a specific element on hover/focus → use `Tooltip` instead (`tooltip.component.spec.md`) |

## 4. Component States

> No local `useState`/`useReducer` in this wrapper. The open/closed axis is owned by Radix's internal `useControllableState` inside `DialogPrimitive.Root`, and is documented here because it is the primary state axis of the compound component (not because our code stores it).

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `idle` (closed) | `open` is `false` / `defaultOpen` `false` (default) | `DialogPortal`/`DialogOverlay`/`DialogPrimitive.Content` are unmounted (Radix uses conditional mounting, not just `hidden`) | none (no dialog in the DOM) |
| `open` | `open` becomes `true` (trigger click, controlled `open` prop, or `DialogTrigger`/programmatic activation) | `DialogOverlay` + `DialogPrimitive.Content` mount; focus moves into the panel (see §9); rest of the page becomes inert if `modal` (default) | full, scoped to the dialog panel |

Radix additionally sets `data-state="open" | "closed"` on `Content`/`Overlay`/`Trigger` — not consumed by any CSS in this wrapper today.

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Always render `DialogTrigger`/`DialogContent`/`DialogClose` as descendants of a single `Dialog` | Render `DialogContent` (or any other part) outside a `Dialog` — Radix's internal context hook throws `` `DialogContent` must be used within `Dialog` `` |
| Use the standalone `DialogClose` export for extra close affordances (e.g. a "Cancelar" button in `DialogFooter`) | Expect to restyle or remove the built-in close (X) icon inside `DialogContent` — it is hardcoded with no prop to disable or configure it |
| Rely on `DialogTitle` for the dialog's accessible name (required for a `role="dialog"` element to be labeled) | Omit `DialogTitle` — Radix logs an accessibility warning if `Content` has no accessible name |

## 8. BDD Scenarios

### Default render

```
Given a Dialog with a DialogTrigger and a DialogContent (containing DialogHeader > DialogTitle and a DialogFooter > DialogClose)
When the trigger is clicked
Then a role="dialog" element becomes visible, containing the title text, with the overlay covering the rest of the page
```

### Closing via the built-in close button

```
Given the Dialog is open
When the user clicks the hardcoded close (X) icon inside DialogContent, or the standalone DialogClose in the footer
Then onOpenChange(false) fires, the dialog and overlay unmount, and focus returns to the DialogTrigger
```

### Keyboard navigation

```
Given the Dialog is open and focus has moved into the panel
When the user presses Esc
Then the dialog dismisses (onOpenChange(false) fires) and focus returns to the element that opened it (the DialogTrigger)
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | `DialogTitle` provides the accessible name via Radix's auto-wired `aria-labelledby` on `DialogPrimitive.Content`; `DialogDescription` (if present) is wired via `aria-describedby`. No `DialogTitle` present means `Content` has no accessible name (Radix warns) |
| Keyboard | `Esc` closes the dialog (Radix `DismissableLayer`); `Tab`/`Shift+Tab` cycle focus only within the panel while open (Radix `FocusScope` trap) |
| Focus management | On open: Radix moves focus into `DialogPrimitive.Content` (first focusable element, or the content itself if none) — overridable via `onOpenAutoFocus`. On close: focus returns to the element that triggered the open (typically `DialogTrigger`) — overridable via `onCloseAutoFocus` |
| ARIA states | `role="dialog"` and `aria-modal="true"` (when `modal`) set by `DialogPrimitive.Content`; `data-state="open" \| "closed"` on `Content`/`Overlay`/`Trigger` |
