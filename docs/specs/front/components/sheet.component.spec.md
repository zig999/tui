## 1. Purpose and Responsibilities

Compound component wrapping `@radix-ui/react-dialog` (`^1.1.19`). Renders a side panel: a full-screen scrim overlay plus a right-anchored, full-height panel (`inset-y-0 right-0`, `max-w-[50vw]`) that traps focus and blocks interaction with the rest of the page until dismissed. Structurally it is a Dialog variant (same Radix primitive) restyled and repositioned as a slide-in panel, plus an optional close-attempt guard not present on `Dialog`.

Exported parts:
- `Sheet` — local wrapper (not a direct re-export) around `DialogPrimitive.Root`. Destructures only `open`, `onOpenChange`, `children` — does **not** spread any other props to `Root`, so `defaultOpen`, `modal`, and any other `Root` prop are unreachable through this wrapper. The panel is always controlled (`open`/`onOpenChange` are both required) and always modal (Radix's `modal` default of `true` applies, with no way to override it).
- `SheetContent` — local wrapper around `DialogPrimitive.Content`. Composition root: renders `DialogPrimitive.Portal` containing `DialogPrimitive.Overlay` (`fixed inset-0 z-40 bg-black/60`) followed by `DialogPrimitive.Content` (fixed right-anchored panel, `z-50`), followed by a close affordance (plain `<button>` or `DialogPrimitive.Close` — see below), followed by `children`.
- `SheetHeader` — local, unstyled `<div>` layout wrapper (`flex items-center gap-3 px-6 pt-6 pb-4`). No Radix primitive underneath.
- `SheetTitle` — local wrapper around `DialogPrimitive.Title` (renders as whatever Radix `Title` renders, semantically a heading); applies `font-semibold tracking-wider text-accent uppercase`.
- `SheetBody` — local, unstyled `<div>` layout wrapper (`flex-1 overflow-y-auto px-6 pb-6`). No Radix primitive underneath.

`SheetHeader` (containing `SheetTitle`) and `SheetBody` are meant to be composed as `children` of `SheetContent`.

**`onCloseAttempt` guard (the behavior distinguishing `Sheet` from `Dialog`):** `SheetContent` accepts an optional `onCloseAttempt?: () => void`. When set:
- `onPointerDownOutside` (backdrop click) and `onEscapeKeyDown` (`Esc`) both call `e.preventDefault()` (blocking Radix's default dismiss) and then call `onCloseAttempt()` instead — the consumer decides whether to actually close (e.g. a dirty-form guard).
- The close (X) affordance is rendered as a **plain `<button>`** (not `DialogPrimitive.Close`) whose `onClick` calls `onCloseAttempt()` only — it does not close the panel itself.

When `onCloseAttempt` is absent, all three dismiss paths (backdrop click, `Esc`, X button) fall through to Radix's default behavior: `DialogPrimitive.Close` is rendered for the X button, and `onPointerDownOutside`/`onEscapeKeyDown` are not intercepted, so Radix calls `onOpenChange(false)` directly.

`SheetContent` also sets `aria-describedby={undefined}` explicitly on `DialogPrimitive.Content`, and there is no `SheetDescription` export — this silences Radix's dev-mode warning about a missing description without providing one.

**Out of scope for this component:**
- Rendering a custom overlay — `SheetContent` always mounts its own `DialogPrimitive.Overlay` inline with a fixed `className`; there is no exported `SheetOverlay` part.
- Configuring `modal`/`defaultOpen` on `Sheet` — the wrapper does not forward any prop beyond `open`/`onOpenChange`/`children`.
- A footer or description slot — unlike `Dialog`, there is no `SheetFooter` or `SheetDescription` export.
- Entry/exit animation on the panel or overlay — no transition/animation classes are present (contrast with `date-picker`, which does animate its popover via `motion/react`). Radix sets `data-state="open" | "closed"` that a consumer could target with custom CSS, but no such CSS exists here.

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Presenting a focused task, detail view, or form in a side panel that must block interaction with the rest of the page until dismissed, especially when the content benefits from full viewport height (e.g. long forms, detail drill-downs) | Presenting a small, centered, non-full-height focused task or confirmation → use `Dialog` instead (`dialog.component.spec.md`) |
| The dismiss action must be interceptable (e.g. "discard unsaved changes?" guard) via `onCloseAttempt` | Showing a transient, non-blocking status message → use `sonner`'s `<Toaster />` instead |

## 4. Component States

> No local `useState`/`useReducer` in this wrapper. The open/closed axis is owned by Radix's internal `useControllableState` inside `DialogPrimitive.Root`, and is documented here because it is the primary state axis of the compound component (not because our code stores it).

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `idle` (closed) | `open` is `false` | `DialogPrimitive.Portal`/`Overlay`/`Content` are unmounted (Radix uses conditional mounting, not just `hidden`) | none (no panel in the DOM) |
| `open` | `open` becomes `true` (consumer sets it, e.g. from a trigger `onClick`) | Overlay (`bg-black/60`, `z-40`) + panel (`inset-y-0 right-0`, `z-50`, `max-w-[50vw]`) mount; focus moves into the panel (see §9); rest of the page becomes inert (always modal) | full, scoped to the panel |

Radix additionally sets `data-state="open" | "closed"` on `Content`/`Overlay` — not consumed by any CSS in this wrapper today.

## 5. Events Emitted

| Event | Payload type | When emitted | Consumer action |
|-------|-------------|--------------|------------------|
| `onOpenChange` | `(open: boolean) => void` | Only when `onCloseAttempt` is **absent**: X button click, `Esc`, or outside pointer-down, each calling `onOpenChange(false)` via Radix's default dismiss path | Consumer updates the state it controls (`open`) |
| `onCloseAttempt` | `() => void` | Only when **provided** to `SheetContent`: X button click, `Esc`, or outside pointer-down — fires **instead of** the default Radix dismiss (which is prevented via `preventDefault()`) | Consumer decides whether to actually close (e.g. show a confirm dialog for unsaved changes, then call `onOpenChange(false)` itself if confirmed) |

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Always render `SheetContent` as a descendant of `Sheet` | Render `SheetContent` outside a `Sheet` — Radix's internal context hook throws |
| Pass `onCloseAttempt` when the panel guards a dirty/unsaved state, and call `onOpenChange(false)` yourself from inside it once confirmed | Assume passing `onCloseAttempt` closes the panel — it only intercepts the dismiss attempt; the consumer must still close explicitly |
| Rely on `SheetTitle` for the panel's accessible name (required for a `role="dialog"` element to be labeled) | Omit `SheetTitle` — Radix logs an accessibility warning if `Content` has no accessible name |
| Use `Sheet` for full-height, right-anchored panels | Reach for `modal={false}` or `defaultOpen` — neither is reachable through `Sheet`'s props; use `Dialog` directly if that control is needed |

## 8. BDD Scenarios

### Default render

```
Given a Sheet with open=false and a SheetContent (containing SheetHeader > SheetTitle and a SheetBody)
When the consumer sets open to true
Then a role="dialog" element becomes visible, right-anchored and full-height, containing the title text, with the bg-black/60 overlay covering the rest of the page
```

### Close-attempt guard (onCloseAttempt set)

```
Given the Sheet is open and SheetContent received an onCloseAttempt callback
When the user presses Esc, clicks the backdrop, or clicks the X button
Then onCloseAttempt fires and onOpenChange is NOT called — the panel stays open until the consumer explicitly changes `open`
```

### Keyboard navigation (no onCloseAttempt)

```
Given the Sheet is open and SheetContent received no onCloseAttempt
When the user presses Esc
Then onOpenChange(false) fires, the panel and overlay unmount, and focus returns to the element that opened it
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | `SheetTitle` provides the accessible name via Radix's auto-wired `aria-labelledby` on `DialogPrimitive.Content`. `aria-describedby` is explicitly set to `undefined` (no `SheetDescription` export exists) |
| Keyboard | `Esc` closes the panel by default, or invokes `onCloseAttempt` instead when set (see §5); `Tab`/`Shift+Tab` cycle focus only within the panel while open (Radix `FocusScope` trap, always active since `Sheet` is always modal) |
| Focus management | On open: Radix moves focus into `DialogPrimitive.Content` (first focusable element, or the content itself if none) — overridable via the forwarded `onOpenAutoFocus`. On close: focus returns to the element that triggered the open — overridable via the forwarded `onCloseAutoFocus` |
| ARIA states | `role="dialog"` and `aria-modal="true"` set by `DialogPrimitive.Content` (always modal); `data-state="open" \| "closed"` on `Content`/`Overlay` |
