## 1. Purpose and Responsibilities

A compound, dependency-free set of thin wrappers around native table elements —
`Table`, `TableHeader`, `TableBody`, `TableRow`, `TableHead`, `TableCell` — each
re-themed to project semantic tokens via `cn()`. `Table` itself renders a
`<div data-slot="table-container" className="w-full overflow-x-auto">`
wrapping the actual `<table>` element, so horizontal scrolling is built in for
narrow viewports; the `className` prop passed to `Table` is applied to the
inner `<table>`, not to the wrapper `div`.

<!-- MIGRATION NOTE (from source): the prior monolithic, columns/data-driven
     Table also pulled in sibling primitives (Empty, Skeleton) for loading/empty
     states. This compound version dropped those affordances along with sorting
     and density controls — they must be composed at the call site if needed. -->

**Out of scope for this component:** data binding (`columns`/`data` props),
sorting, pagination, row selection, loading state, empty state — none of these
exist in this version; consumers compose plain JSX rows/cells directly.

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Rendering static or manually-mapped tabular data (rows known ahead of render, no built-in sort/paginate needed) | Data needs sorting, pagination, filtering, or row selection → compose a `@tanstack/react-table`-backed table instead (not part of this primitive) |
| A simple key/value or list-like grid with a header row | Content needs virtualization for very large row counts (>~1000 rows) → this component has no virtualization |

## 6. Variants and Compositions

> No `variant`/`size` prop exists on any of the six pieces — there is no `cva()` block in this component. All styling is a fixed set of Tailwind classes per element, merged with the consumer's `className` via `cn()`. This table documents the compound composition instead.

| Component | Rendered element(s) | Fixed token classes |
|-----------|---------------------|----------------------|
| `Table` | `<div data-slot="table-container">` + `<table data-slot="table">` | wrapper: `w-full overflow-x-auto` · table: `w-full border-collapse text-sm` |
| `TableHeader` | `<thead data-slot="table-header">` | `border-b border-border bg-surface` |
| `TableBody` | `<tbody data-slot="table-body">` | none beyond consumer `className` |
| `TableRow` | `<tr data-slot="table-row">` | `border-b border-border odd:bg-transparent even:bg-zebra transition-colors hover:bg-hover motion-reduce:transition-none` |
| `TableHead` | `<th data-slot="table-head">` | `h-10 px-3 text-left align-middle text-xs font-medium uppercase tracking-wider text-muted-foreground` |
| `TableCell` | `<td data-slot="table-cell">` | `px-3 py-2 align-middle text-foreground` |

Zebra striping is applied via `bg-zebra` on even-indexed `TableRow` elements
(`even:bg-zebra`), and the header row separator is `border-b` on
`TableHeader`. Both are unconditional — no prop toggles them off.

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Compose `TableHeader`/`TableBody`/`TableRow`/`TableHead`/`TableCell` manually around your own mapped data | Don't expect a `columns`/`data`-driven API, built-in sorting, pagination, loading skeleton, or empty state — all were dropped in the migration to this compound shape |
| Use `className` on `TableCell`/`TableHead` for column-specific alignment (e.g. `text-right`) | Don't assume `TableHead` sets `scope="col"` — it does not (see §9) |

## 8. BDD Scenarios

### Default render

```
Given Table/TableHeader/TableBody/TableRow/TableHead/TableCell are composed
  with a header row and N data rows
When the tree mounts
Then a native <table> renders inside a horizontally-scrollable wrapper,
  with the header row bordered by border-b and bg-surface
```

### Key state — zebra striping

```
Given a TableBody containing multiple TableRow children
When it renders
Then every even-indexed TableRow receives a bg-zebra background (odd rows
  stay transparent) purely from static Tailwind classes — no prop controls it
```

### Keyboard navigation

```
Given a table whose TableHead/TableCell content is plain text (no interactive
  children)
When the user presses Tab
Then focus skips over the table entirely — no element in this compound
  component sets tabIndex or a keydown handler
```
