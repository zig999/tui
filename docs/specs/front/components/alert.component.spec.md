## 1. Purpose and Responsibilities

A terminal-styled inline alert: a `border-l-2` accent, a `lucide-react` icon,
an optional bold title, body content, an optional action slot, and an
optional self-dismiss button. Dismissal is fully local â€” the component
unmounts itself (`return null`) once dismissed; there is no callback to
notify the consumer.

**Out of scope for this component:** persistence of dismissal across
remounts, toast/notification stacking (see `sonner` for that), and notifying
the consumer when the alert is dismissed (no `onDismiss` prop exists â€” see
Â§4 and the note below).

| Use when | Do not use when |
|----------|-----------------|

| The message should be dismissible by the user without consumer-side state tracking | The consumer needs to know *when* the alert was dismissed (e.g., to log, persist, or clear related state) â†’ not supported; no `onDismiss` callback exists in this source |

## 6. Variants and Compositions

Base class applied unconditionally: `relative flex gap-3 border-l-2 bg-surface p-4`.

| Variant | Prop | Border token | Icon (`lucide-react`) | Text/title color token | Default `role` | Usage context |
|---------|------|----------------|--------------------------|---------------------------|-------------------|----------------|
| `info` | `variant="info"` (default) | `border-info` | `Info` | `text-info` | `status` | Neutral/contextual information |
| `success` | `variant="success"` | `border-success` | `CheckCircle2` | `text-success` | `status` | Confirmation of a successful operation |
| `warning` | `variant="warning"` | `border-warning` | `AlertTriangle` | `text-warning` | `alert` | Cautionary message requiring attention |
| `destructive` | `variant="destructive"` | `border-destructive` | `XCircle` | `text-destructive` | `alert` | Error / failure message |

The variant icon is rendered `mt-0.5 size-4 shrink-0` with `aria-hidden="true"`.
The dismiss button (when `dismissible`) always uses the `X` icon at
`size-3.5`, independent of `variant`.

## 8. BDD Scenarios

### Default render

```
Given an Alert with no variant prop (defaults to "info") and a title + children
When it mounts
Then it renders with role="status", the Info icon (text-info, aria-hidden),
  the title in text-info, and border-info on the left edge
```

### Key state â€” dismissed

```
Given an Alert with dismissible=true
When the user clicks the dismiss button (aria-label "Fechar aviso")
Then the internal dismissed state becomes true and the component renders
  null â€” its previously-visible content (e.g. the title text) is no longer
  in the document
```

### Keyboard navigation

```
Given a dismissible Alert is rendered
When the user presses Tab to focus the dismiss button, then presses Enter
  or Space (native <button> activation)
Then the alert dismisses identically to a mouse click
```
