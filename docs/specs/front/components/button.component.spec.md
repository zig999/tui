## 1. Purpose and Responsibilities

Renders a clickable action trigger (native `<button>` by default, or wraps a single child element via Radix `Slot` when `asChild` is true). Applies the TUI visual language (1px box border, sharp corners, uppercase tracked label, invert-on-hover) through five `variant` styles and four `size` scales, and manages a `loading` presentation (spinner + `aria-busy` + forced disable) on top of the native `disabled` state.

**Out of scope for this component:** form submission logic, navigation logic (when used as a link via `asChild`, the wrapped element owns `href`/routing), and click debouncing/loading-state ownership — `loading` is fully controlled by the consumer (no internal timer or async handling).

---

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Triggering an action (submit, confirm, open, delete) or, via `asChild`, styling a single interactive child (e.g. a router `Link`) as a button | Navigating as a plain inline text link with no button semantics → use the `Link` component (`link.component.spec.md`) instead |

## 3. Props Contract

```ts
import type { ComponentProps } from "react";
import type { VariantProps } from "class-variance-authority";
import type { buttonVariants } from "./button";
```

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `variant` | `"primary" \| "secondary" \| "ghost" \| "destructive" \| "outline"` (from `VariantProps<typeof buttonVariants>`) | no | `"primary"` | Visual style variant (see §6) |
| `size` | `"sm" \| "md" \| "lg" \| "icon"` (from `VariantProps<typeof buttonVariants>`) | no | `"md"` | Size scale (see §6) |
| `asChild` | `boolean` | no | `false` | Renders the Radix `Slot` instead of a `<button>`, merging props onto the single child element |
| `loading` | `boolean` | no | `false` | Shows a spinning `Loader2` icon before `children` and forces the disabled state (`disabled ?? loading`) plus `aria-busy` |
| `disabled` | `boolean` (native) | no | `undefined` | Native disabled attribute; effective disabled state is `disabled ?? loading` |
| `className` | `string` | no | `undefined` | Merged with the variant classes via `cn()` |
| ...props | `ComponentProps<"button">` | no | — | + native `<button>` attributes (e.g. `onClick`, `type`, `aria-label`, `form`) |

## 4. Component States

> No internal state (no `useState`/`useReducer`). All states below are prop-driven presentation the component computes on every render.

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `idle` | initial render, `loading=false`, `disabled` falsy | variant/size classes applied, no spinner | full |
| `loading` | `loading === true` | prepends an animated `Loader2` icon (`h-4 w-4 animate-spin`, `motion-reduce:animate-none`, `aria-hidden="true"`) before `children`; element receives `disabled` and `aria-busy="true"` | none (pointer-events disabled via `disabled:pointer-events-none` on the effective disabled state) |
| `disabled` | `disabled === true` (and not overridden by `loading`) | `disabled:opacity-50 disabled:pointer-events-none` applied | none |

## 6. Variants and Compositions

| Variant | Prop | Token classes | Usage context |
|---------|------|----------------|----------------|
| `primary` | `variant="primary"` (default) | `border-primary bg-primary text-primary-foreground hover:bg-transparent hover:text-primary` | <!-- TO CONFIRM: primary/main call-to-action --> |
| `secondary` | `variant="secondary"` | `border-border bg-surface text-foreground hover:border-border-strong hover:bg-elevated hover:text-primary` | <!-- TO CONFIRM: secondary action --> |
| `ghost` | `variant="ghost"` | `border-transparent bg-transparent text-foreground hover:bg-hover hover:text-primary` | <!-- TO CONFIRM: low-emphasis / toolbar action --> |
| `destructive` | `variant="destructive"` | `border-destructive bg-destructive text-destructive-foreground hover:bg-transparent hover:text-destructive` | <!-- TO CONFIRM: destructive/irreversible action --> |
| `outline` | `variant="outline"` | `border-border bg-transparent text-foreground hover:border-primary hover:text-primary` | <!-- TO CONFIRM: outlined secondary emphasis --> |

| Size | Prop | Token classes |
|------|------|----------------|
| `sm` | `size="sm"` | `h-7 px-2 text-xs` |
| `md` | `size="md"` (default) | `h-9 px-3 text-sm` |
| `lg` | `size="lg"` | `h-11 px-4 text-base` |
| `icon` | `size="icon"` | `h-9 w-9 p-0` |

**Base classes (all variants/sizes):** `inline-flex items-center justify-center gap-2 border font-medium tracking-wider uppercase transition-colors outline-none select-none focus-visible:ring-2 focus-visible:ring-ring disabled:pointer-events-none disabled:opacity-50`

**Default variants:** `variant: "primary"`, `size: "md"`

**`asChild` composition:** when `true`, renders `@radix-ui/react-slot`'s `Slot` instead of `<button>`, merging `data-slot="button"`, the computed className, `disabled`, `aria-busy`, and all passthrough props onto the single child element.

---

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Pass `aria-label` when using `size="icon"` with non-text content (see `IconOnly` story) | Rely on `loading` to also debounce/queue repeated clicks — it only disables the element, it does not track request state |
| Use `asChild` to compose a single interactive child (e.g. router `Link`) with Button's visual styling | Wrap multiple children under `asChild` — Radix `Slot` requires exactly one child element |

## 8. BDD Scenarios

### Default render

```
Given the component receives no variant/size props
When it mounts
Then it renders a <button> with variant="primary" size="md" classes and no spinner
```

### Loading state

```
Given loading is true
When it mounts
Then the button renders a spinning Loader2 icon before children, is disabled, and has aria-busy="true"
```

### Keyboard navigation

```
Given the button is focused (not disabled, not loading)
When the user presses Enter or Space
Then the native button activates and onClick fires; focus-visible ring (focus-visible:ring-ring) is shown per the base classes
```

### Disabled — no click

```
Given loading is true (button is disabled via disabled ?? loading)
When the user attempts to click the button
Then onClick does not fire and the button remains in the disabled/aria-busy state
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | Native text content via `children`; for icon-only buttons (`size="icon"`) the consumer must pass `aria-label` explicitly (component does not infer one) |
| Keyboard | Native `<button>` semantics — Enter/Space activate; when `asChild` is used, keyboard behavior is inherited from the wrapped element |
| Focus management | `focus-visible:ring-2 focus-visible:ring-ring` on keyboard focus; no focus trapping or redirection (single interactive element) |
| ARIA states | `aria-busy="true"` when `loading`; effective `disabled` attribute is `disabled ?? loading` |
