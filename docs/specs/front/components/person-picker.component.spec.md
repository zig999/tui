## 1. Purpose and Responsibilities

A single-person picker: a `button[aria-haspopup="listbox"]` trigger showing the selected person's avatar (or bordered initials-box fallback) and name, or a placeholder. When open, renders a search input plus a `ul[role="listbox"]` of candidates filtered by the typed query. Selection is fully controlled (`value: string | null` / `onChange`); the component holds no selected-person state of its own, only the open/closed flag, the typed search query, and the keyboard-focused index. It always narrows `people` locally by `name` (client-side fallback filter) and optionally calls `onSearch(query)` on every keystroke so a caller can drive server-side search in parallel.

**Out of scope for this component:** fetching/paginating people (caller supplies the full `people` array via props), per-person custom colors (source had a colored circular avatar with an arbitrary hex color per person — removed; replaced by a bordered initials box using semantic tokens only), and multi-selection (see `multi-combobox`).

**Migration note:** the source component already received `people` as a prop and made no `@tanstack/react-query` calls internally — no data-fetching code needed removal for this migration. What changed: `PersonOption` (`id`/`name`/`initials`/`color`) → `PersonPickerOption` (`id`/`name`/`avatarUrl?`), with initials now derived from `name`; the inline "clear" control moved from a nested `span[role="button"]` inside the trigger `<button>` (invalid nested-interactive-control pattern) to a sibling `<button>`.

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| The user must choose exactly one person from a filterable list, shown with avatar/initials and name | More than one person must be selectable → use `multi-combobox` instead <!-- TO CONFIRM: multi-combobox works on generic {value,label,sublabel} options, not {id,name,avatarUrl} people — confirm whether a multi-person variant is planned or multi-combobox is expected to be reused with mapped options --> |
| Options fit in memory as a flat array (caller may still narrow the array server-side via `onSearch`) | The picker needs to show avatars for a non-person entity → use a generic option-based primitive instead |

## 3. Props Contract

```ts
import type {
  PersonPickerOption,
  PersonPickerProps,
```

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `people` | `PersonPickerOption[]` | yes | — | `{ id: string; name: string; avatarUrl?: string }[]` — full candidate list; caller owns fetching/filtering upstream (see §3.1). |
| `value` | `string \| null` | yes | — | Selected person `id`, or `null` when nothing is selected. Matched against `people[].id` to resolve the displayed person (see §3.1). |

| `placeholder` | `string` | no | `"Selecionar pessoa..."` | Trigger text shown when no person is selected. |
| `disabled` | `boolean` | no | `false` | Disables the trigger button (`pointer-events-none cursor-not-allowed opacity-50`); hides the clear button even if a person is selected. |
| `className` | `string` | no | — | Merged onto the root `div` via `cn()`. |
| `ref` | `Ref<HTMLDivElement>` | no | — | Forwarded to the root `div` (normal prop, React 19 — no `forwardRef`); manually assigned alongside the component's own `containerRef` via an internal `assignRef` helper. |
| — | + native `div` attributes (via `ComponentProps<"div">`, minus `value`/`onChange`) | no | — | Spread onto the root `div`, e.g. `aria-label`, `id`. |

## 3.1 Data Contract

**Field usage:**

| Field | Source prop | Internal use | Behavior if absent |
|-------|-------------|--------------|-------------------|
| `id` | `PersonPickerOption.id` (each item in `people`) | Identity key for `role="option"` items; compared against the top-level `value: string \| null` to resolve the selected person and to mark `aria-selected` | Required field — `PersonPickerOption` has no optional `id` |
| `name` | `PersonPickerOption.name` | Rendered as the trigger text (when selected) and the option row text; also the field the local client-side filter matches against (`name.toLowerCase().includes(search)`); also the source for computed initials (first letter of first word + first letter of last word, uppercased; `"?"` if the name has no non-whitespace parts) | Required field |
| `avatarUrl` | `PersonPickerOption.avatarUrl` | When present, renders an `<img>` (`h-8 w-8 border border-border object-cover`, `alt=""` decorative); when absent, renders the bordered initials box instead | Falls back to the initials box (`initialsOf(name)`) |

**Cross-prop join rules:**

| Prop A | Field A | Prop B | Field B | Relationship |
|--------|---------|--------|---------|--------------|
| `value` | (top-level string \| null) | `people` | `PersonPickerOption.id` | `matches on` — the component derives `selectedPerson = people.find(p => p.id === value) ?? null` to render the trigger content; no match (or `value === null`) renders the placeholder |

## 4. Component States

> Internal state: `isOpen` (boolean), `search` (string), `focusedIndex` (number, keyboard-focused option). A `containerRef` (outside-click detection) and `searchRef`/`listRef` (autofocus + scroll-into-view) are refs, not rendered state.

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `idle` (closed) | initial render / dropdown closed | Trigger shows selected person's avatar+name or the placeholder; `border-border`; `aria-expanded="false"` | full |
| `open` | click on trigger (`open()`, no-op if `disabled`) | Trigger gets `border-primary`; a search input + `ul[role="listbox"]` mount below (`absolute z-20 mt-1 ... border border-border bg-elevated`); search input auto-focuses after a `setTimeout(0)`; `search` resets to `""`, `focusedIndex` resets to `-1` | full |
| `focused-index` | `ArrowDown`/`ArrowUp` in the search input, or mouse-enter on an option | The option at `focusedIndex` **or** the option matching `value` gets `bg-primary text-primary-foreground`; `scrollIntoView({ block: "nearest" })` runs on change | full |
| `loading` | `loading === true` while dropdown is open | List shows a single `role="presentation"` row with a spinning `Loader2` (`animate-spin`, `motion-reduce:animate-none`) and "Carregando..." instead of options | none (list not interactive while shown) |
| `empty-results` | dropdown open, not loading, `filtered.length === 0` | A `role="presentation"` row shows "Nenhum resultado encontrado." | none |
| `closed-on-outside-interaction` | `mousedown` outside `containerRef` (document listener), `Escape`, or `Tab` while the search input is focused | Dropdown unmounts; `search` resets to `""`, `focusedIndex` resets to `-1` | full |
| `disabled` | `disabled === true` | Trigger gets `pointer-events-none cursor-not-allowed opacity-50`; clear button is not rendered even if a person is selected | none |

> **Transition parameters:**

| Parameter | Formula / Value | Unit | Applies to state |
|-----------|----------------|------|-----------------|
| Dropdown offset | `mt-1` | rem | `open` |
| Dropdown max height | `max-h-52` (13rem) | rem | `open` |
| Search-input autofocus delay | `setTimeout(..., 0)` | ms | `open` |
| Avatar / initials box size | `h-8 w-8` (2rem) | rem | all |
| Clear icon size | `h-4 w-4` (1rem) | rem | all |
| Loading spinner size | `h-3.5 w-3.5` (0.875rem) | rem | `loading` |

## 5. Events Emitted

| Event | Payload type | When emitted | Consumer action |
|-------|-------------|--------------|-----------------|
| `onChange` | `(id: string \| null) => void` | An option is selected (mousedown on an option, or `Enter` while `focusedIndex` points to one) → emits that option's `id`, then the dropdown closes; or the clear button is clicked → emits `null` (dropdown state unaffected) | Update the controlled `value` prop to the emitted id or `null` |
| `onSearch` | `(query: string) => void` | Every keystroke in the search input | Optionally refetch/narrow `people` server-side for the new query |

## 6. Variants and Compositions

| Variant | Prop | Usage context |
|---------|------|----------------|

| `trigger-disabled` | `disabled === true` | `pointer-events-none cursor-not-allowed opacity-50` |

| `option-active` | `focusedIndex === index \|\| id === value` | `bg-primary text-primary-foreground` |
| `option-default` | *(not active)* | `text-foreground hover:bg-hover` |

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Provide a stable `value: string \| null` and update it from `onChange` — the component is fully controlled and holds no selection state | Rely on the component to fetch or paginate `people` — it is purely presentational; fetch in a `features/*/api/` hook and pass the result down |
| Pass `avatarUrl` when a photo is available; omit it to get the initials-box fallback automatically | Expect the clear control to be reachable while `disabled` — it is not rendered in that state |
| Pair with `onSearch` when the candidate set must be re-fetched per keystroke | Assume `onSearch` disables the built-in local name filter — both run; `onSearch` is additive |

## 8. BDD Scenarios

### Default render

```
Given the component receives valid required props (value: null, people, onChange)
When it mounts
Then the trigger renders closed, showing the placeholder text, aria-expanded="false", no clear button
```

### Loading state

```
Given loading is true
When the trigger is clicked to open the dropdown
Then the list shows a single row with a spinning Loader2 icon and "Carregando..." instead of any options
```

### Keyboard navigation

```
Given the dropdown is open and the search input is focused
When the user presses ArrowDown/ArrowUp
Then focusedIndex moves (wrapping at the ends) among the currently filtered people and the focused option is highlighted and scrolled into view
When the user then presses Enter
Then onChange is called with the focused person's id and the dropdown closes
When the user presses Escape or Tab
Then the dropdown closes without calling onChange
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Label | No built-in visible `<label>`; component provides no fallback `aria-label` on the trigger — consumer must supply one or an associated external `<label>` |
| Keyboard | `ArrowDown`/`ArrowUp` move `focusedIndex` in the search input; `Enter` selects the focused option; `Escape` and `Tab` close the dropdown; clicking outside the root container also closes it |
| Focus management | On open: DOM focus moves to the search input after a `setTimeout(0)`; options are `<li>` elements, not natively focusable — "focused" is a visual highlight only (`focusedIndex` state), not a DOM focus move; on close (Escape/Tab/outside click/selection): no explicit focus-return to the trigger is coded <!-- TO CONFIRM: no programmatic focus return to the trigger button on close; confirm whether this is an acceptable deviation from the full ARIA combobox pattern --> |
| ARIA states | Trigger: `aria-haspopup="listbox"`, `aria-expanded`; list: `role="listbox"`; each item `role="option"` with `aria-selected` (true when `id === value`); avatar/initials box and `<img>` are `aria-hidden="true"` / `alt=""` (decorative, name text carries the accessible content); no `aria-activedescendant` linking the search input to the keyboard-focused option |
