## 1. Purpose and Responsibilities

Fully custom compound component — **does not wrap any Radix primitive** (no `@radix-ui/react-tabs` dependency in the package; confirmed absent from `package.json`). Selection state is shared between parts via a `React.Context` (`TabsContext`) created at module scope, not via a Radix scope.

Exported parts:
- `Tabs` — root. Holds the active-tab state (own `useState`) and provides `{ value, onValueChange }` to descendants via `TabsContext`. Renders a `<div className="flex flex-col">`.
- `TabsList` — renders a `<div role="tablist" className="flex gap-0 border-b border-border">`.
- `TabsTrigger` — renders a `<button role="tab">`. Reads `TabsContext`, computes `isSelected = activeValue === value` itself, and calls `onValueChange(value)` on click. Renders a `▸` marker and applies underline/color styling when selected; optionally renders a `[N]` count badge when the `count` prop is set.
- `TabsContent` — renders a `<div role="tabpanel">` for its own `value`, or **returns `null`** (fully unmounts, not just visually hides) when its `value` does not match the active tab.

`useTabsContext()` throws `Error("Tabs subcomponent used outside <Tabs>")` if any subcomponent using it (`TabsTrigger`, `TabsContent`) renders without an ancestor `Tabs`.

**Out of scope for this component:**
- Keyboard arrow-key navigation between tabs — **no `onKeyDown` handler exists anywhere in this file**. Only pointer `onClick` on `TabsTrigger` switches tabs; native `Tab`/`Shift+Tab` browser focus traversal is the only keyboard interaction implemented (see §9).
- Preserving inactive tab content in the DOM — `TabsContent` unmounts (`return null`) rather than hiding via CSS, so any local state inside an inactive tab's content is lost when switching away and re-mounted fresh on return.
- Orientation (`aria-orientation`) — not set; layout is a fixed horizontal row (`flex`).

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Switching between a small, fixed set of content panels within the same page context, where all tab labels are known at compose-time (authored as `TabsTrigger` children) | Full-page/route-level navigation is needed → use `@tanstack/react-router` routes instead |
| | Standards-compliant `ArrowLeft`/`ArrowRight` roving-tab keyboard navigation is a hard requirement → this implementation has no arrow-key handling (see §1, §9); <!-- TO CONFIRM: whether this is an accepted gap or must be fixed before wider adoption --> |

## 3. Props Contract

### `Tabs` (root)

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `defaultValue` | `string` | yes | — | Seeds the internal `useState` as the initially active tab's `value` |
| `value` | `string` | no | `undefined` (uncontrolled) | Controlled active-tab override. Resolved value is `controlledValue ?? internalValue` |
| `onValueChange` | `(value: string) => void` | no | — | Called after the internal state updates, with the newly selected tab's `value` |
| `className` | `string` | no | `undefined` | Merged via `cn()` onto the root `<div>` (default `flex flex-col`) |
| ...props | `ComponentProps<"div">` (minus `defaultValue`/`value`/`onValueChange`, which are own props) | no | — | + native `<div>` attributes |

**Internal behavior note (from code, not paraphrased):** `handleValueChange` **always** calls `setInternalValue(newValue)`, even when `value` is controlled — the controlled prop simply takes precedence in `value = controlledValue ?? internalValue` while it stays defined. There is no `useEffect` syncing `internalValue` to a changing `value` prop. <!-- TO CONFIRM: whether this is an accepted implementation detail or an actual bug — a consumer setting `value` back to `undefined` after controlling it could see a stale internalValue reappear -->

### `TabsList`

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| ...props | `ComponentProps<"div">` | no | — | + native `<div>` attributes. Component fixes `role="tablist"` |
| `className` | `string` | no | `undefined` | Merged via `cn()` with `flex gap-0 border-b border-border` |

### `TabsTrigger`

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `value` | `string` | yes | — | This tab's identity. Compared against the context's active value to compute `isSelected`; passed to `onValueChange` when clicked |
| `count` | `number` | no | `undefined` | When set, renders a `[N]` badge after `children`, with `aria-label={`${count} itens`}` on the badge `<span>` |
| `children` | `ReactNode` | no | — | Tab label content |
| `className` | `string` | no | `undefined` | Merged via `cn()` onto the `<button>` |
| ...props | `ComponentProps<"button">` (minus `value`, which is destructured as an own prop) | no | — | + native `<button>` attributes — **including `onClick` and `type`, both of which are overridden** if the consumer passes them, because `{...props}` is spread in JSX *after* the component's own hardcoded `onClick={() => onValueChange(value)}` and `type="button"` (see Do/Don't, §7) |

Component always sets `role="tab"`, `aria-selected={isSelected}`, `tabIndex={isSelected ? 0 : -1}` (roving tabindex) — all of which are also overridable by a matching key in `...props` for the same reason.

### `TabsContent`

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `value` | `string` | yes | — | This panel's identity. Compared against the context's active value; if it doesn't match, the component returns `null` (unmounts) |
| `children` | `ReactNode` | no | — | Panel content, rendered only while active |
| `className` | `string` | no | `undefined` | Merged via `cn()` with `pt-4` |
| ...props | `ComponentProps<"div">` (minus `value`) | no | — | + native `<div>` attributes. Only applied when the panel is active (`role="tabpanel"` is fixed) |

**Context (internal, not a public prop):** `TabsContextValue = { value: string; onValueChange: (value: string) => void }`. Provided by `Tabs` via `TabsContext` (`React.Context` created at module scope). `TabsTrigger`/`TabsContent` read it via `useTabsContext()`, which **throws** `Error("Tabs subcomponent used outside <Tabs>")` if there is no ancestor `Tabs`.

## 4. Component States

> `Tabs` owns one real `useState` (`internalValue`). `TabsTrigger`/`TabsContent` have no `useState` of their own — their selected/active states are computed on every render from context, and listed here because each component sets the resulting attribute itself (`aria-selected`, `tabIndex`, or unmounting).

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `idle` (initial) | `Tabs` mounts with `defaultValue` | The `TabsTrigger` matching `defaultValue` renders selected (see below); its `TabsContent` renders, all others are unmounted | full |
| `TabsTrigger` — selected | `activeValue === trigger.value` | `border-primary text-primary`, `▸` marker rendered before `children`, `aria-selected="true"`, `tabIndex={0}` | full (tabbable) |
| `TabsTrigger` — unselected | `activeValue !== trigger.value` | `border-transparent text-muted-foreground` (hover → `text-foreground`), no `▸` marker, `aria-selected="false"`, `tabIndex={-1}` | full (clickable, but not in the native `Tab` sequence) |
| `TabsContent` — active | `activeValue === content.value` | Renders its `children` inside `role="tabpanel"` | full |
| `TabsContent` — inactive | `activeValue !== content.value` | Component returns `null` — fully unmounted, not just hidden | none (not in the DOM) |

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Always render `TabsTrigger`/`TabsContent` as descendants of a `Tabs` | Render `TabsTrigger` or `TabsContent` outside a `Tabs` — `useTabsContext()` throws `Error("Tabs subcomponent used outside <Tabs>")` |
| Give every `TabsTrigger` a `value` that has a matching `TabsContent` | Pass a custom `onClick` (or `type`) to `TabsTrigger` expecting it to run *in addition to* the built-in tab switch — it **replaces** the internal `onClick={() => onValueChange(value)}` because `{...props}` spreads after it in JSX, silently breaking tab switching |
| Control `value`/`onValueChange` from the parent when the active tab must be synced elsewhere (e.g. the URL) | Assume `ArrowLeft`/`ArrowRight` move focus/selection between tabs — no keyboard handler for arrow keys exists; only `Tab` (native roving tabindex) and click switch tabs |

## 8. BDD Scenarios

### Default render

```
Given a Tabs with defaultValue="profile" and three TabsTrigger/TabsContent pairs (profile, activity, settings)
When it mounts
Then the "profile" trigger has aria-selected="true" and shows the ▸ marker, its TabsContent renders, and the other two TabsContent panels are not in the DOM
```

### Switching tabs on click

```
Given the Tabs above is rendered with "profile" active
When the user clicks the "activity" TabsTrigger (which has count={3})
Then onValueChange("activity") fires, the "activity" trigger's aria-selected becomes "true", its TabsContent mounts and renders, and the "profile" TabsContent unmounts
```

### Keyboard navigation

```
Given the Tabs is rendered with "profile" active (tabIndex=0) and the other triggers have tabIndex=-1
When the user presses Tab
Then focus moves to the next focusable element after the active TabsTrigger in the page's native tab order — no arrow-key handling exists to move focus/selection between TabsTrigger elements
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | `TabsTrigger`'s visible `children` is its accessible name (native `<button>` text content); no separate `aria-label` is set by the component |
| Keyboard | Native `<button>` semantics only — `Enter`/`Space` activate a focused `TabsTrigger` (fires its `onClick`); **no custom `ArrowLeft`/`ArrowRight` handling** to move focus between tabs |
| Focus management | Roving `tabIndex` (`0` for the selected trigger, `-1` for the rest) — only the active tab is in the native `Tab` sequence; no focus is moved programmatically on tab switch |
| ARIA states | `role="tablist"` on `TabsList`; `role="tab"` and `aria-selected` on `TabsTrigger`; `role="tabpanel"` on the active `TabsContent` (inactive panels are absent from the DOM, not `aria-hidden`) |
