## 1. Purpose and Responsibilities

Renders a native `<kbd>` element styled as a small keycap-style label for representing a single keyboard key or shortcut symbol inline (e.g. `Ctrl`, `K`). Two `size` values (`sm`/`md`) control padding only; both sizes render the same `text-xs` font size.

**Out of scope for this component:** composing multiple keys into a shortcut string (e.g. "Ctrl+K") — the consumer composes multiple `<Kbd>` elements plus a separator manually (see the `Shortcut` story: `<Kbd>Ctrl</Kbd>` + `<span aria-hidden="true">+</span>` + `<Kbd>K</Kbd>`); the component itself renders exactly one key label per instance.

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Representing a literal keyboard key or symbol inline (a shortcut hint, a single key label) | The content is a multi-word instruction or full sentence → use plain text instead, `Kbd` is reserved for literal key labels/symbols |

## 3. Props Contract

```ts
import type { ReactNode } from "react";
```

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `children` | `ReactNode` | yes | — | Key label or symbol rendered inside the `<kbd>` element. |
| `size` | `"sm" \| "md"` | no | `"md"` | Controls padding only: `sm` → `px-1 py-0.5`, `md` → `px-1.5 py-1`. Font size is `text-xs` for both sizes — the source's original 10px/11px arbitrary sizes were normalized to the single `text-xs` token and now differ only by padding (see MIGRATION comment in code). |
| `className` | `string` | no | `undefined` | Merged via `cn()` onto the `<kbd>` element. |

Note: `KbdProps` is a plain interface (does not extend `ComponentProps<"kbd">`) — the component destructures only `{ children, size, className }`; no native `<kbd>` attribute passthrough is spread.

## 4. Component States

> No internal `useState`/`useReducer`, and no state-like attribute is computed by the component beyond the size-driven padding class selection documented in §6 — no internal state.

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `idle` | initial render | renders `<kbd className="inline-flex items-center justify-center border border-border bg-elevated text-xs text-accent select-none">{children}</kbd>` plus the size-driven padding classes (§6) | none — not interactive |

## 6. Variants and Compositions

No `cva()` block exists — the two size values are applied via inline conditional class expressions (`size === "sm" && "px-1 py-0.5"`, `size === "md" && "px-1.5 py-1"`) rather than a `cva()` `variants` map, and there is no `defaultVariants` object (the `"md"` default comes from the destructuring default `size = "md"`).

| Size | Prop | Token classes |
|------|------|----------------|
| `sm` | `size="sm"` | `px-1 py-0.5` (+ base classes) |
| `md` | `size="md"` (default) | `px-1.5 py-1` (+ base classes) |

**Base classes (both sizes):** `inline-flex items-center justify-center border border-border bg-elevated text-xs text-accent select-none`

---

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Wrap each physical key in its own `<Kbd>` and join multiple keys with a separate `aria-hidden` separator (see the `Shortcut` story: `Ctrl` + `+` + `K`) | Put a multi-key shortcut string as a single `children` value inside one `<Kbd>` — each key should be its own instance |
| Use plain key labels/symbols as `children` (e.g. `"Ctrl"`, `"K"`) | Use `Kbd` for sentences or instructional text — it is not a general-purpose inline badge |

## 8. BDD Scenarios

### Default render

```
Given a Kbd with children="K" and no size prop
When it mounts
Then it renders a <kbd> with the md padding classes (px-1.5 py-1) plus the base classes
(border border-border bg-elevated text-xs text-accent select-none)
```

### Small size state

```
Given a Kbd with children="K" and size="sm"
When it mounts
Then it renders the sm padding classes (px-1 py-0.5) instead of md padding,
with font size still text-xs (unchanged between sizes)
```

### Keyboard navigation

```
Given a Kbd is rendered on a page with other focusable elements (e.g. inside the Shortcut composition)
When the user tabs through the page
Then the Kbd element is never focused — it sets no tabIndex, no interactive role, and no key handlers;
select-none additionally prevents its text from being selected, since it represents a key label rather
than an interactive control
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | No `aria-label` is set internally — the accessible name comes from `children` text content inside the native `<kbd>` element |
| Keyboard | Not focusable/interactive — no `tabIndex` or key handlers; `select-none` prevents text selection |
| Focus management | None — the component never receives or manages focus |
| ARIA states | None set internally — relies on the native `<kbd>` element's implicit semantics |
