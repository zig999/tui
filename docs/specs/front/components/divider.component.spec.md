## 1. Purpose and Responsibilities

Renders a dashed separator rule in one of three mutually exclusive forms, selected by an if/else chain evaluated on every render: a vertical rule (`vertical === true`), a labeled horizontal rule with centered caption text (`label` set, and `vertical` falsy), or a plain horizontal `<hr>` (neither set). `vertical` is checked first and returns immediately, so a `label` passed together with `vertical={true}` is silently ignored.

**Out of scope for this component:** orientation auto-detection (no container-based switching between horizontal/vertical — `vertical` is an explicit boolean prop), and multiple/rich label content (`label` is a single `string`, not `ReactNode`).

---

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Visually separating two sections of content with a horizontal or vertical rule, optionally with a centered caption (e.g. an "OR" divider between two form options) | Only spacing is needed with no visual rule → use Tailwind gap/space utilities directly instead of this component |

---

## 3. Props Contract

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `label` | `string` | no | `undefined` | When set **and** `vertical` is falsy, renders the labeled variant: two `flex-1` dashed rules flanking an uppercase, tracked, muted-foreground label span. Silently ignored if `vertical` is `true` (the vertical branch returns before the label check runs). |
| `vertical` | `boolean` | no | `false` | When `true`, renders a vertical dashed rule (`<span>` sized `h-full w-px`) instead of a horizontal one; takes precedence over `label`. |
| `className` | `string` | no | `undefined` | Merged via `cn()` onto whichever root element the active branch renders (`<span>`, `<div>`, or `<hr>`). |

Note: `DividerProps` is a plain interface (does not extend any native element's `ComponentProps`) — the component destructures only `{ label, vertical, className }`, no native attribute passthrough is spread onto the rendered element.

---

## 4. Component States

> No internal `useState`/`useReducer`. The three rows below are mutually exclusive render branches the component computes from `vertical` and `label` on every render, with `vertical` checked first.

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|----------------|
| `default` (horizontal, unlabeled) | `vertical` falsy and no `label` | renders `<hr aria-hidden="true" className="border-dashed border-border">` | none — decorative |
| `labeled` | `label` set and `vertical` falsy | renders `<div role="separator" aria-label={label}>` containing two `flex-1 border-t border-dashed border-border` spans flanking a `shrink-0 text-xs tracking-wide text-muted-foreground uppercase` label span | none — structural |
| `vertical` | `vertical === true` (any `label` value is ignored) | renders `<span aria-hidden="true" className="inline-block h-full w-px border-l border-dashed border-border">` | none — decorative |

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Use `label` alone (with `vertical` unset/false) to get the centered-caption horizontal rule | Pass both `vertical` and `label` expecting the label to render — `vertical` short-circuits before the label branch and `label` is silently dropped |
| Place a `vertical` Divider inside a flex row where the parent provides an explicit height (the rule uses `h-full`) | Use this component for pure layout spacing with no visual rule — use gap/space utilities instead |

## 8. BDD Scenarios

### Default render

```
Given a Divider with no props
When it mounts
Then it renders a <hr aria-hidden="true"> with classes border-dashed border-border
```

### Labeled state

```
Given a Divider with label="OR" (vertical unset)
When it mounts
Then it renders a <div role="separator" aria-label="OR"> with two flex-1 dashed rules
flanking an uppercase "OR" span, and no <hr> is rendered
```

### Vertical overrides label (key state)

```
Given a Divider with vertical={true} and label="OR" both set
When it mounts
Then only the vertical <span aria-hidden="true" h-full w-px border-l> is rendered,
and the "OR" label is not rendered anywhere
```

### Keyboard navigation

```
Given a Divider (in any of its three forms) is rendered on a page with other focusable elements
When the user tabs through the page
Then the Divider is never focused — none of its three markup forms sets tabIndex,
a role, or a keyboard handler that would make it an interactive stop
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | `aria-label={label}` is set only on the `labeled` branch's `role="separator"` `<div>`; the `default` `<hr>` and `vertical` `<span>` carry no label |
| Keyboard | Not focusable/interactive in any branch — no `tabIndex`, no key handlers |
| Focus management | None — purely decorative/structural, never receives or manages focus |
| ARIA states | `role="separator"` only on the `labeled` branch; `aria-hidden="true"` on the `default` `<hr>` and the `vertical` `<span>` (excluded from the accessibility tree) |
