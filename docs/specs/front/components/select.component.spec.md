## 1. Purpose and Responsibilities

A custom (non-native, non-Radix) single-select dropdown. Renders a `button[role="combobox"]` trigger showing the selected option's label (or a placeholder) and, when open, a `ul[role="listbox"]` of options rendered from the `options` array. Selection is fully controlled: the component holds no selected value of its own, only the open/closed flag and the keyboard-highlighted index.

**Out of scope for this component:** multi-selection (see `multi-combobox`), free-text filtering/search of options, per-option custom colors (an earlier source variant supported an arbitrary CSS color per option — removed to keep to semantic tokens only), an `Empty`-state sibling component (a plain muted-text row is rendered inline instead when `options` is empty).

---

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Choosing exactly one value from a list of options in a compact, single-line trigger | All options must stay visible at once for direct comparison → use `radio-group` instead |
| The option list can be flat, unfiltered, and rendered as plain text labels | The user needs to select more than one option → use `multi-combobox` instead |
| The option count is small enough to scroll in a `max-h-60` panel without in-list search | The option set is large and needs text filtering → use `multi-combobox` instead <!-- TO CONFIRM: multi-combobox is the only other list-selection primitive in the kit; confirm it is the intended filtering alternative --> |

## 3. Props Contract

```ts
import type { SelectOption, SelectProps } from "@/shared/components/ui/select/select.types";
```

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `value` | `string \| null` | yes | — | Currently selected option value; `null` renders the placeholder. Matched against `options[].value` to resolve the displayed label (see §3.1). |
| `onChange` | `(value: string) => void` | yes | — | Called with the selected option's `value` when the user picks a non-disabled option. |
| `options` | `SelectOption[]` | yes | — | `{ value: string; label: string; disabled?: boolean }[]` — the full list rendered in the dropdown. |
| `placeholder` | `string` | no | `"Selecionar..."` | Text shown in the trigger when `value` is `null` or matches no option. |
| `disabled` | `boolean` | no | `false` | Disables the trigger button (`pointer-events-none opacity-50`) and blocks all keyboard/mouse handlers. |
| `className` | `string` | no | — | Merged onto the root `div` via `cn()`. |
| `ref` | `Ref<HTMLDivElement>` | no | — | Forwarded to the root `div` (normal prop, React 19 — no `forwardRef`). |
| — | + native `div` attributes (via `ComponentProps<"div">`, minus `value`/`onChange`) | no | — | Spread onto the root `div`, e.g. `aria-label`, `id`. |

## 3.1 Data Contract

**Field usage:**

| Field | Source prop | Internal use | Behavior if absent |
|-------|-------------|--------------|-------------------|
| `value` | `SelectOption.value` (each item in `options`) | Rendered as the option's identity in `role="option"` list items and compared against the top-level `value` prop | Required field — `SelectOption` has no optional `value` |
| `label` | `SelectOption.label` | Rendered as the visible text in both the trigger (when selected) and the option row | Required field |
| `disabled` | `SelectOption.disabled` | Marks the option `aria-disabled`, applies `opacity-40 cursor-not-allowed`, excludes it from keyboard `ArrowUp`/`ArrowDown` navigation and blocks selection via click or `Enter` | Treated as `false` — option is selectable |

**Cross-prop join rules:**

| Prop A | Field A | Prop B | Field B | Relationship |
|--------|---------|--------|---------|--------------|
| `value` | (top-level string) | `options` | `SelectOption.value` | `matches on` — the component derives `selectedOption = options.find(o => o.value === value)` to render the trigger label; no match renders the placeholder |

## 4. Component States

> No `useReducer`; two `useState` hooks (`isOpen`, `focusedIndex`) plus `checked`/`disabled`-equivalent behavior is prop-driven (see §3).

| State | Trigger | Visual change | Interactivity |
|-------|---------|---------------|---------------|
| `idle` (closed) | initial render / dropdown closed | Trigger shows selected label or placeholder; `border-border`, chevron not rotated; `aria-expanded="false"` | full |
| `open` | click on trigger, or `ArrowDown`/`ArrowUp`/`Enter` while closed (and not `disabled`) | Trigger gets `border-primary`, chevron rotates 180°, `aria-expanded="true"`; `ul[role="listbox"]` mounts (`absolute z-20 mt-1 max-h-60 ... border border-border bg-elevated`) | full |
| `focused-index` | `ArrowDown`/`ArrowUp` while open, or mouse-enter on a non-disabled option | The option at `focusedIndex` gets `bg-primary text-primary-foreground` (and its check icon, if selected, switches to `text-primary-foreground`); `scrollIntoView({ block: "nearest" })` runs on change | full |
| `closed-on-outside-interaction` | `mousedown` outside the root container (document listener), `Escape`, or `Tab` while open | Dropdown unmounts, `focusedIndex` resets to `-1` | full |

> **Transition parameters:**

| Parameter | Formula / Value | Unit | Applies to state |
|-----------|----------------|------|-----------------|
| Dropdown max height | `max-h-60` (15rem) | rem | `open` |
| Dropdown offset | `mt-1` | rem | `open` |
| Icon size (chevron/check) | `14` | px | all |

| Event | Payload type | When emitted | Consumer action |
|-------|-------------|--------------|-----------------|
| `onChange` | `(value: string) => void` | User clicks (`mousedown`) a non-disabled option, or presses `Enter` while a non-disabled option is keyboard-focused (`focusedIndex`) | Update the controlled `value` prop to the emitted string |

> No `cva()` call exists in `select.tsx` — there is no `variant`/`size` prop. Visual differentiation is entirely state-driven (internal state, see §4) or prop-driven (`disabled`, see §3). Token classes per state, for reference:

| Variant | Prop | Usage context |
|---------|------|---------------|
| `trigger-closed` | *(internal `isOpen === false`)* | `border-border hover:border-border-strong bg-elevated text-foreground` |
| `trigger-open` | *(internal `isOpen === true`)* | `border-primary` |
| `trigger-disabled` | `disabled === true` | `pointer-events-none opacity-50` |

| `option-focused` | *(internal `focusedIndex === index`)* | `bg-primary text-primary-foreground cursor-pointer` |
| `option-disabled` | `SelectOption.disabled === true` | `opacity-40 cursor-not-allowed` |
| `option-selected` (check icon) | `SelectOption.value === value` | Renders `Check` icon, `text-primary` (or `text-primary-foreground` if also focused) |

## 7. Do / Don't

| Do | Don't |
|----|-------|
| Provide a stable `value` (or `null`) and update it from `onChange` — the component is fully controlled | Don't expect the component to track its own selection; it holds no selected-value state |
| Mark unselectable entries with `SelectOption.disabled` | Don't remove disabled options from the array if they must still be visible-but-inert; the component already excludes them from keyboard nav and click |
| Pass `aria-label` (or wrap with an associated `<label>`) since the trigger has no visible `<label htmlFor>` wiring | Don't rely on placeholder text alone for a11y — it is not an accessible name |

## 8. BDD Scenarios

### Default render

```
Given the component receives valid required props (value: null, options, onChange)
When it mounts
Then the trigger renders closed, showing the placeholder text, aria-expanded="false"
```

### Disabled option cannot be selected

```
Given an option in `options` has disabled: true
When the user clicks that option or keyboard-navigates to it and presses Enter
Then onChange is not called and the dropdown does not close from that action
```

### Keyboard navigation

```
Given the trigger is focused and the dropdown is closed
When the user presses ArrowDown
Then the dropdown opens and focusedIndex is set to 0 (or, if already open, presses Enter)
Then onChange is called with the focused option's value, the dropdown closes, and focusedIndex resets to -1
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Label | No built-in visible `<label>`; consumer must supply `aria-label` or an external `<label>` referencing the trigger (see story's `"aria-label": "Status"` arg) |
| Keyboard | `ArrowDown`/`ArrowUp` open the dropdown and move `focusedIndex` (skipping disabled options); `Enter` opens the dropdown or selects the focused option; `Escape` and `Tab` close it |
| Focus management | DOM focus never leaves the trigger `<button>` — options are `<li>` elements, not natively focusable; the "focused" option is a visual highlight only (`focusedIndex` state), not a DOM focus move. <!-- TO CONFIRM: no `aria-activedescendant` links the trigger to the visually-highlighted option; confirm if this is an acceptable deviation from the full ARIA combobox pattern --> |
| ARIA states | `role="combobox"` + `aria-expanded` + `aria-haspopup="listbox"` on the trigger; `role="listbox"` on the panel; each item `role="option"` with `aria-selected` and `aria-disabled` |
