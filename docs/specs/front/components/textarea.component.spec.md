## 1. Purpose and Responsibilities

Renders a styled native `<textarea>` primitive: a box-bordered, vertically-resizable multi-line text field consuming semantic tokens (`border-border`, `bg-background`, `text-foreground`, `placeholder:text-muted-foreground`), with reactive styling for the `disabled` and `aria-invalid` native attributes.

**Out of scope for this component:** label rendering, character counter, and helper/error text. Per the source's MIGRATION note, the original component bundled a composite field (optional label header, char counter, helper/error text); this was simplified to a bare `<textarea>` primitive, consistent with `Input` ‚Äî label/counter/helper text must compose from separate primitives around this one.

## 2. When to Use / When Not to Use

| Use when | Do not use when |
|----------|-----------------|
| Capturing multi-line free-text input | Capturing a single line of text ‚Üí use `Input` instead |

## 4. Component States

> No internal state (no `useState`/`useReducer`). The component does not set `disabled` or `aria-invalid` itself ‚Äî both are native attributes the consumer passes through `...props`; the rows below document the CSS reaction the component's stylesheet applies when the consumer sets them.

| State | Trigger | Visual change | Interactivity |
|-------|---------|----------------|----------------|
| `idle` | initial render, no `disabled`/`aria-invalid` set | base classes only (`border-border`, minimum height `min-h-16`, vertical resize handle) | full |
| `invalid` | consumer sets `aria-invalid="true"` (or `true` boolean) | `aria-invalid:border-destructive` ‚Äî border switches to the destructive token | full (field remains interactive) |
| `disabled` | consumer sets `disabled` | `disabled:cursor-not-allowed disabled:opacity-50` | none |

## 6. Variants and Compositions

No `variant`/`size` props and no `cva()` ‚Äî single visual style (consistent with the Component Contract: CVA is used only for 2+ variants).

**Base classes (single style, applied unconditionally):** `flex min-h-16 w-full resize-y border border-border bg-background px-3 py-2 text-sm text-foreground placeholder:text-muted-foreground outline-none transition-colors focus-visible:ring-2 focus-visible:ring-ring motion-reduce:transition-none aria-invalid:border-destructive disabled:cursor-not-allowed disabled:opacity-50`

`resize-y` permits the consumer/user to vertically resize the field via the native textarea resize handle; horizontal resize is not enabled.

| Do | Don't |
|----|-------|

| Set `aria-invalid` to drive the destructive border styling | Use this component for single-line input ‚Üí use `Input` |

## 8. BDD Scenarios

### Default render

```
Given the component receives a placeholder and no other props
When it mounts
Then it renders a native <textarea> with min-h-16, resize-y, and base token classes with no error/disabled styling
```

### Invalid state

```
Given aria-invalid is set to true
When it mounts
Then the textarea's border switches to the destructive token (aria-invalid:border-destructive) while remaining focusable and editable
```

### Keyboard navigation

```
Given the textarea is focused
When the user types multi-line characters
Then the value updates, onChange fires for each keystroke, and the focus-visible ring (focus-visible:ring-ring) remains visible while focused
```

## 9. Accessibility Contract

| Requirement | Implementation |
|-------------|-----------------|
| Label | Not rendered by this component ‚Äî wraps a labeled native `<textarea>`; consumer must associate an external `<label htmlFor>` |
| Keyboard | Native `<textarea>` semantics ‚Äî multi-line text entry, Tab to focus/blur; no custom key handlers |
| Focus management | `focus-visible:ring-2 focus-visible:ring-ring` on keyboard focus; no focus trapping |
| ARIA states | `aria-invalid` supported via native passthrough and styled with `aria-invalid:border-destructive`; consumer is responsible for `aria-describedby` linking to an error message |
